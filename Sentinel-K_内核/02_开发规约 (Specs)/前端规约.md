# 网页与插件开发规约 (Plugin & Frontend Specs)

> **用途**: AI 编码时的核心开发约束清单，包含 Obsidian 插件特有的沙盒限制与样式隔离规范。

---

## 🛑 核心红线 (必须遵守)

### 1. 移动端沙盒与文件 I/O 限制
- ❌ **严禁使用 Node.js 核心 API**: 禁止引入 `fs`、`path`、`os` 等 Node 原生包（这些包在 iOS/Android 移动端沙盒环境下会导致插件直接崩溃）。
- ✅ **强制使用抽象适配层**: 所有文件读取、写入、重命名、判断存在操作，必须全面通过 Obsidian 官方提供的底座接口进行：
  - `this.app.vault.adapter.read(...)`
  - `this.app.vault.adapter.write(...)`
  - `this.app.vault.adapter.rename(...)`
  - `this.app.vault.adapter.exists(...)`
- ✅ **相对路径规范**: 传递给 `adapter` 的所有路径必须相对于 Vault 根目录，且必须使用 `app.vault.adapter.normalizePath(path)` 进行路径标准化，防止跨平台斜杠 `\` 与 `/` 差异。

### 2. 界面弹窗与提示
- ❌ **严禁使用外部重型 UI 组件**: 禁止引入 Element Plus, Ant Design, Vuetify 等重型 UI 库（这会导致 main.js 包体积暴增且样式不协调）。
- ❌ **禁止原生 alert/confirm**: 禁止调用原生 `window.alert()` / `window.confirm()`。
- ✅ **原生通知**: 状态通知一律使用 Obsidian 原生 `new Notice(message, duration)`。
- ✅ **轻量级交互**: 在 Vue 视图中，使用微型 Vanilla CSS 编写交互组件，或使用 Obsidian 的原生 `Modal` 进行确认弹窗。

### 3. CSS 样式污染与隔离
- ✅ **全局高亮类名前缀**: 为防止样式污染 Obsidian 软件自身的 DOM，所有在 `styles.css` 中定义的全局样式必须使用特定命名空间前缀 `lang-learner-`。
  - 例如高亮包裹类：`.lang-learner-word`、`.lang-learner-word.un-known`。
- ✅ **Vue 组件样式隔离**: 在 `Panel.vue` 等单文件组件中，必须在 `<style>` 标签上添加 `scoped` 属性：
  ```html
  <style scoped>
  /* 仅在组件内部生效的样式 */
  </style>
  ```
- ❌ **禁止内联样式**: 严禁在 HTML / Vue 模板中拼写超长内联 `style`，动态样式（如位置计算）除外。

### 4. 异常处理与中文日志
- ✅ **异常必捕获**: 所有异步操作（特别是文件 I/O、网络翻译请求、DOM 拦截解析）必须使用 `try-catch` 块包裹。
- ✅ **中文日志与报错**: 在代码中生成的所有 `console.log`、`console.error`、`throw new Error(...)` 以及 `Notice` 提示信息，必须使用 **中文（简体）**。

### 5. Vue 3 规范
- ✅ **`<script setup>` 语法**: 所有 `.vue` 文件必须使用 Vue 3 组合式 API (Composition API) 与 `<script setup lang="ts">`。
- ✅ **解耦与状态通信**: 由于 Obsidian 会存在多标签页，状态变化后必须通过 Event Bus 广播 `lang-learner:word-changed` 事件，各 UI 自行响应，禁止强依赖父子组件引用。

## 📂 项目推荐目录结构

```text
src/
├── db/              # 本地影子词库与数据存储控制 (Memory-First DB)
├── tokenizer/       # 文本清理、分词与词干还原模块 (Lemmatizer)
├── ui/              # Obsidian View 挂载类与 Vue 组件
│   ├── SidebarView.ts
│   └── Panel.vue
├── event/           # 跨标签页及 UI 解耦的 Event Bus
├── generator/       # 语境卡片 (.md) 笔记生成逻辑
├── types.ts         # TypeScript 类型定义 (WordStatus, WordInfo 等)
├── main.ts          # 插件主入口文件 (Plugin Lifecycle)
└── styles.css       # 插件全局 CSS 与零 JS 功耗 hover 气泡样式
```

---

## 命名规范

- Vue 组件: 大驼峰命名法 `Panel.vue`。
- 逻辑脚本: 小驼峰命名法 `lemmatizer.ts`。
- CSS 类名: 全小写，中划线分隔，并带上前缀，如 `lang-learner-word`。

---

## Git 提交规范

格式: `<type>(<scope>): <subject>`

Type 类型:
- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 重构
- `docs`: 文档更新
- `style`: 样式调整
- `perf`: 性能优化
